# Freelance Playbook - Prvi projekti

Kratki operativni dokument za vođenje freelance projekta od prvog razgovora do handovera.

## 1. Discovery pitanja (prvi sastanak)

- Koji je glavni poslovni cilj projekta (vise rezervacija, manje ručnog rada, bolja vidljivost)?
- Što je "must-have" za launch, a što može ići u fazu 2?
- Tko su korisnici i kako trenutno rade rezervacije?
- Postoji li postojeći sustav (myrent, rent-a-car app, CRM) i imamo li API pristup?
- Tko s klijentove strane daje tehničke pristupe i odobrava isporuku?
- Koji je okvirni budžet i željeni rok?

## 2. Scope template (što ulazi / ne ulazi)

### Uključeno

- Frontend stranice i osnovni UX flow prema dogovoru
- Backend API sloj i integracija s vanjskim sustavom (ako su pristupi dostupni)
- Osnovno testiranje ključnih korisničkih tokova
- Deploy i inicijalna produkcijska konfiguracija

### Nije uključeno (osim ako se posebno ugovori)

- Dodatne integracije izvan inicijalnog opsega
- Napredni admin/CMS panel
- SEO strategija i marketing kampanje
- Višejezični sadržaj iznad dogovorenog broja jezika
- Neograničen broj revizija nakon odobrenog scopea

## 3. Pricing guardrails

- Definiraj "walk-away" brojku prije pregovora.
- Ako klijent ne može platiti puni iznos, smanji scope, ne cijenu.
- Koristi faze: MVP (launch) -> faza 2 (nadogradnje).
- Uvijek napiši što uzrokuje rast cijene (integracije, revizije, nejasni zahtjevi, promjena scopea).

## 4. Ugovorne klauzule (minimum)

- Opseg i isporuke (deliverables)
- Rokovi i odgovornosti obje strane
- Plaćanje po fazama (npr. 40% / 40% / 20%)
- Ownership i prava korištenja nakon pune uplate
- Third-party ovisnosti (API dostupnost, SaaS outage) i ograničenje odgovornosti
- Promjene opsega (change request) i kako se dodatno naplaćuju
- Podrška nakon isporuke (trajanje i što uključuje)

## 5. Handover checklist

- Git repozitorij i pristupi
- Hosting/Render pristupi i environment varijable
- Domena i DNS pristupi
- API ključevi (myrent i ostali)
- Osnovna tehnička dokumentacija (run/deploy, ključni tokovi)
- Backup i rollback napomena
- Kontakt točka za post-launch period

## 6. Operativni ritam za svaki projekt

1. Discovery poziv
2. Scope + procjena + ponuda
3. Ugovor i avans
4. Implementacija po fazama
5. Demo i feedback
6. Finalna isporuka i handover
7. Podrška po ugovoru

## 7. Kratke rečenice za pregovore

- "Mogu prilagoditi opseg fazno, ali za puni traženi opseg ne mogu ići ispod minimalne dogovorene cijene."
- "Ako budžet treba optimizirati, predlažem MVP sada i nadogradnje u fazi 2."
- "Da bismo držali rok i kvalitetu, trebamo zaključati scope prije početka razvoja."
