1. O projektu

Ovaj prijedlog nastaje kao odgovor na Vašu potrebu za modernom web stranicom koja posjetiteljima omogućuje pregled smještaja i vozila, online rezervaciju te automatsku sinkronizaciju rezervacija s aplikacijom myrent.

Radi bržeg i transparentnijeg dogovora, djelomično funkcionalan prototip (PoC) stranice već je izrađen i dostupan za demonstraciju. Posjetitelji mogu pretraživati ponudu, ispuniti rezervacijski obrazac i obaviti online prijavu gostiju.

Cilj ovog razgovora jest razumjeti Vaše poslovne potrebe, provjeriti usklađenost prototipaom s Vašim očekivanjima i dogovoriti sljedeće korake.

2. Trenutni status – što je već gotovo

Prototip stranice uključuje sljedeće funkcionalne komponente:

  • Naslovna stranica s navigacijom i brendiranjem GOTOVO
  • Pregled smještajnih jedinica (placeholder sučelje) U TIJEKU
  • Pregled i pretraživanje rent-a-car vozila GOTOVO
  • Automatski prikaz apartmana i cijena iz myrent API-ja U TIJEKU
  • Implementacija rent-a-car rezervacija (ovisno o dostupnosti postojećeg sustava) PLANIRA SE
  • Inline obrazac za rezervaciju uz validaciju GOTOVO
  • Online prijava gostiju (eVisitor-ready) GOTOVO
  • REST API za kreiranje rezervacija (pohrana u JSON) GOTOVO
  • Stranice: Aktivnosti, FAQ, Kako do nas, O nama GOTOVO
  • Simulirana potvrda rezervacije e-mailom U TIJEKU
  • myrent-kompatibilne API putanje (local adapter) GOTOVO
  • Spajanje na produkcijski myrent API (API ključevi klijenta) U TIJEKU
  • Pravi sadržaj (tekst, fotografije, cijene) KLIJENT DOSTAVLJA

3. Integracija s myrent sustavom

myrent ima službenu API dokumentaciju (myrent.readme.io) i potvrđene endpointove za web integraciju. Prototip je već pripremljen tako da koristi iste putanje i strukturu zahtjeva kao myrent API, uz lokalni adapter za demonstraciju bez API ključeva.

  • Model rada: myrent-first (myrent je glavni izvor podataka za objekte, cijene i dostupnost)
  • Podrzane myrent putanje: customer/list, customer/prices_fast/{id}, booking/calendar/{id}
  • Podrzano kreiranje rezervacije: rents/add/{object_id}
  • Lokalni adapter koristi placeholder myrent ID-eve i lokalne JSON datoteke za PoC fazu
  • Produkcijsko spajanje aktivira se nakon unosa MYRENT_B2B_GUID i MYRENT_USER_GUID
  • Zastita stabilnosti: cache + fallback prikaz zadnje valjane verzije podataka ako je API privremeno nedostupan

* Za finalno produkcijsko povezivanje potrebni su API pristupni podaci klijenta i potvrda automatski dohvaćenog mapiranja objekata prije objave.

5. Okvirna investicija

Cijena se formira prema konačnom opsegu, no okvirni rasponi su sljedeći:

Izrada i prilagodba stranice (finalni UX, sadržaj, responsive) 1.200 – 2.500 EUR
Automatski prikaz i osvježavanje podataka iz myrent API-ja 300 – 700 EUR
myrent API integracija i mapiranje ID-eva 600 – 1.300 EUR
Rent-a-car implementacija (integracija postojećeg sustava ili novi modul) 500 – 2.000 EUR
Sinkronizacijski sloj (cache/fallback, timeout/retry, error log) 250 – 700 EUR
Hosting i domena (godišnje) 50 – 120 EUR / god.
Podrška i održavanje (opcija) 30 – 80 EUR / mj.
Okvirni ukupni projekt 2.850 – 6.600 EUR

* Konačna cijena dogovara se nakon definiranja opsega. Navedene su neto cijene.

6. Što je potrebno s Vaše strane

  • MYRENT_B2B_GUID i MYRENT_USER_GUID (ili administratorski pristup za generiranje)
  • Potvrda automatski dohvaćene liste objekata i njihove vidljivosti na web stranici
  • Dogovor oko finalnog UX-a za stranicu smještaja (placeholder prema produkcijskom sučelju)
  • Za rent-a-car: API dokumentacija i pristup postojećem sustavu, ili odobrenje za razvoj novog modula
  • Fotografije smještajnih jedinica i vozila (minimalno 3–5 po jedinici)
  • Opisi, cijene i uvjeti rezervacije
  • Logo i eventualne smjernice brenda (boje, font)
  • Podaci za e-mail potvrde (naziv tvrtke, kontakt podaci)
  • Pristup domeni ili dogovor oko hosting rješenja

7. Pitanja za naš razgovor

  • Koliko smještajnih jedinica i vozila treba biti prikazano?
  • Imate li aktivan API pristup i mozete li odmah dostaviti MYRENT_B2B_GUID i MYRENT_USER_GUID?
  • Za rent-a-car: postoji li već aplikacija s API pristupom ili želite novi modul u sklopu weba?
  • Tko vodi cjenik i dostupnost vozila (postojeći sustav ili nova administracija)?
  • Primaju li se rezervacije samo direktno ili i putem Booking.com / Airbnb?
  • Treba li integracija eVisitor prijave gostiju?
  • Postoji li postojeća domena ili treba nova?
  • Tko će unositi i ažurirati sadržaj nakon objave (Vi ili ja)?
  • Postoji li vizualni identitet (logo, boje) koji treba slijediti?
