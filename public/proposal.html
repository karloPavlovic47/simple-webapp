Prijedlog suradnje (inicijalni razgovor)

Ova verzija služi za prvi sastanak i pregled opsega. Finalna ponuda i cijena definiraju se nakon usklađivanja svih zahtjeva i tehničkih pristupa.

Što je cilj projekta

  • Web prezentacija smještaja i rent-a-car ponude s modernim UX-om.
  • Direktne rezervacije kroz web uz povezivanje s myrent sustavom.
  • Jedno centralno mjesto održavanja podataka (myrent-first model).

Trenutni status

  • Funkcionalan PoC je već izrađen i dostupan za demonstraciju.
  • Smještaj i rent-a-car sučelja postoje s osnovnim rezervacijskim tokom.
  • Dodana je detaljna stranica apartmana (slike, opis, rezervacija).
  • myrent-kompatibilne API putanje su pripremljene (local adapter + API način rada).

Predviđeni opseg implementacije

  • API dokumentacija (provjera endpointa): myrent.readme.io/reference/web-api-guide
  • Spajanje na produkcijski myrent API (MYRENT_B2B_GUID + MYRENT_USER_GUID).
  • Automatski prikaz objekata, cijena, dostupnosti i sadržaja iz myrent sustava.
  • Strict booking način rada za izbjegavanje overbookinga.
  • Stabilnost sustava: cache/fallback prikaz ako je API privremeno nedostupan.
  • Rent-a-car implementacija: Scenarij A: integracija s postojećim sustavom (ako postoji API pristup). Scenarij B: novi modul rent-a-car rezervacija unutar web aplikacije.

Što je potrebno s Vaše strane

  • Pristupni podaci za myrent API (B2B GUID i User GUID).
  • Potvrda koji objekti i vozila idu javno na web.
  • Potvrda scenarija za rent-a-car (postojeći sustav ili novi modul).
  • Finalna potvrda sadržaja i vizualnog smjera prije objave.

Sljedeći koraci

  1. Prezentacija PoC-a i inicijalni razgovor.
  2. Potvrda opsega i tehničkih preduvjeta.
  3. Dostava API pristupa i dogovor modela rent-a-car implementacije.
  4. Dostava finalne ponude s cijenom i rokovima.

Napomena: ovo je inicijalni dokument za prvu fazu razgovora. Cijena nije javno prikazana u ovoj verziji.
